# Contributing Guidelines

We welcome contributions that improve security, automation, or documentation. Please follow these steps:

1. Fork the repository and create a feature branch.
2. Write clear and descriptive commit messages.
3. Ensure code passes tests and linting (CI will run automatically).
4. Submit a pull request with a description of changes.

## Code of Conduct

All contributors are expected to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
